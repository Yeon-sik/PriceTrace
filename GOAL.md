# GOAL

## 제품 가격 관측·비교 플랫폼

사람이 구매할 수 있는 대상의 가격 변화를, 출처가 확인 가능한 관측 데이터로 축적·비교하는 플랫폼을 만든다.

초기에는 영수증 기반 소매 상품에서 시작한다. 이후 식당 메뉴, 원자재, 부동산, 서비스까지 같은 탐색·가격 관측 구조로 확장한다.

## 핵심 원칙

1. 영수증 JSON은 변경하지 않는 구매 원본이다.
2. 판매처별 상품번호는 `판매처 + 상품번호` 범위에서만 식별자로 취급한다.
3. 매장별 판매 상품과 통합 표준 상품을 분리한다.
4. 동일 상품 비교는 표준 상품 ID를 기준으로 한다.
5. 구매 대상 유형과 카테고리 계층을 분리한다.
6. 카테고리는 고정 레이어 컬럼이 아닌 부모-자식 트리로 관리한다.
7. 가격은 실시간 시세가 아니라 관측 시점·단위·위치가 있는 관측가다.
8. 원본 구매 사실, 사용자 정산 상태, 표준 카탈로그, 관리자 검토 이력은 별도로 저장한다.
9. 자동 매핑·자동 분류는 신뢰도와 검토 상태를 남기며, 확정 데이터로 과장하지 않는다.
10. 사용자별 원본 데이터는 RLS로 격리한다. 사용자 간 통합 비교는 명시적 동의와 검증된 집계 경로가 마련된 뒤에만 연다.

## 목표 데이터 흐름

```text
영수증 원본
  → 판매처별 상품 식별
  → 표준 상품 매핑
  → 구매 대상 유형
  → 카테고리 트리
  → 가격 관측 이력
  → 판매처·기간·단위 기준 비교
```

## 구매 대상 유형

- `retail_product`: 소매 상품
- `menu_item`: 식당 메뉴
- `raw_material`: 원자재
- `property`: 부동산
- `service`: 서비스

## 제품 계층

```text
구매 대상 유형
  → 카테고리
    → 하위 카테고리
      → 표준 상품
        → 판매처별 상품번호
          → 가격 관측
```

예: `소매 상품 → 식품 → 가공식품 → 냉동식품 → 표준 상품 → 매장별 상품번호 → 영수증 관측가`

## 현재 우선순위

1. 영수증 기반 정산과 가격 관측의 신뢰성 유지
2. 표준 상품·판매처별 상품 매핑·카테고리 탐색 기반 구축
3. 관리자 검토와 감사 로그로 매핑 품질 관리
4. 표준 상품 기준의 가격 비교 화면 제공
5. 이후 식당 메뉴·원자재·부동산 유형 확장

## 비목표

- 관측되지 않은 가격을 실시간 시세처럼 표현하지 않는다.
- 상품명만으로 동일 상품을 확정하지 않는다.
- 영수증 원본 JSON에 분류·정규화 정보를 덮어쓰지 않는다.
- 사용자 동의 없이 개인 영수증을 다른 사용자에게 공개하지 않는다.
