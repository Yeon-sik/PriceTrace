# PriceTrace

영수증에서 확인된 구매 가격을 기록하고, 같은 판매처·공통 카탈로그·공식 상품 연결의 신뢰 수준에 맞춰 가격 이력을 보여 주는 개인용 가격 관측 도구다. 가격은 실시간 시세가 아니라 **영수증 관측가**다.

최종 갱신: 2026-07-23

## 현재 제공 기능

- `receipt.v2` 범용 영수증 JSON 검증과 로컬 로드
- 판매처, PX/일반 마트, 카테고리, 검색 조건의 상품 목록
- 영수증 관측가와 판매처별 가격 이력
- 장바구니, 공동구매 수량 배분, 친구별 정산 및 localStorage 저장
- 공식 상품 검색 후보와 사용자 확인 기반 연결
- Zod 검증, TypeScript strict, Vitest 도메인 테스트

## 영수증 모델

모든 영수증은 `receipt.v2`를 사용한다. 원본 문서·판매처·품목·합계·결제 정보를 분리해 보존하며, 금액은 KRW 정수로 처리한다.

판매처에는 다음 두 값을 명시한다.

- `merchant.retail_channel`: `px`, `regular`, `unknown`
- `merchant.catalog_namespace`: 검증된 공통 상품 코드 체계. 확인할 수 없으면 `null`

현재 두 PX 영수증은 `korean-military-px` 카탈로그를 사용한다.

## 상품 동일성 및 가격 비교 규칙

상품명만으로는 품목을 식별하지 않는다. 병합과 가격 이력의 기준은 다음과 같다.

| 상황 | 목록 표시와 가격 이력 |
| --- | --- |
| 일반 판매처 상품, 공식 연결 없음 | 같은 판매처의 정규화된 상품명 관측만 함께 본다. |
| 검증된 공통 카탈로그 | 같은 `catalog_namespace`, 상품 코드, 정규화된 상품명이 모두 일치할 때만 서로 다른 매장을 하나의 품목으로 합친다. |
| 공식 상품 연결됨 | 동일한 공식 상품 URL로 확인된 판매처 품목을 하나의 공식 상품으로 묶고, 각 판매처의 관측가를 함께 표시한다. |
| 코드가 같지만 카탈로그가 다름 | 자동 병합하지 않는다. 코드 충돌 가능성이 있으므로 별도 공식 연결 또는 검토가 필요하다. |
| 이름이 같지만 용량·규격이 다름 | 자동 병합하지 않는다. 각 용량은 별도 공식 상품 연결 대상으로 유지한다. |

공통 PX 카탈로그의 같은 코드·같은 이름은 공식 상품 연결 전에도 하나의 카드로 표시하며, 판매처와 관측 횟수를 합산한다. 화면에서는 `공통 카탈로그 상품`으로 구분한다.

## 데이터 경계

- 실제 영수증은 `private-data/`에 보관하고 Git 및 공개 번들에서 제외한다.
- `data/demo/`에는 공개 가능한 비식별 샘플만 둔다.
- 영수증 원본은 UI에서 수정하지 않는다.
- 친구, 배분, 전달·입금 상태는 영수증 원본과 분리해 localStorage에 저장한다.
- 공식 상품 연결은 사용자가 확인한 제조사·공식몰 URL을 기준으로 저장한다.

## 다음 확장 시 원칙

1. OCR은 `receipt.v2` 형식으로만 출력하고, 추측한 상품 정규화는 자동 확정하지 않는다.
2. 공식 상품 연결은 자동 후보 제시까지만 허용하고, 최종 연결은 검증 가능해야 한다.
3. 판매처가 다른 품목을 비교하려면 공통 카탈로그 일치 또는 검증된 공식 상품 연결이 필요하다.
